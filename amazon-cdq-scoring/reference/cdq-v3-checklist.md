## 一、政策违规检测（U 级风险，优先扫描）

### 1.1 索引/网站抑制
- [ ] ASIN 是否被搜索索引 (Is Search Indexed)
- [ ] ASIN 是否在网站活跃 (Is Website Active)
- [ ] 是否处于 Quarantined 状态

### 1.2 标题政策违规
- [ ] 标题是否完整
- [ ] 是否存在标题冗余（关键词堆砌）
- [ ] 是否包含受限关键词
- [ ] 长度或大小写是否错误

### 1.3 敏感词/违禁词扫描

#### 健康与医疗声明
- Antimicrobial, Antibacterial, Antifungal
- Prevents bacteria, Resists mildew, Removes mold
- Sanitizes, Disinfects, Sterilizes
- Anti-Mites, Detoxify
- UV, UV sterilizes, Bacteria, Pesticides

#### 营销与促销词汇
- Gift, Free, Warranty, Guarantee
- Best, Best Seller, Top Rated, #1 Best
- Free Shipping, Cheapest, Discount, Sale
- 100% Quality Guaranteed

#### 品牌侵权类
- 竞品品牌名称
- 通用产品侵权词
- 未授权的品牌引用

#### 引导评论与联系方式
- Review request 相关引导词
- 电话号码、邮箱、网址
- 联系方式

#### 格式与字符限制
- 特殊字符：! $ ? { } _ ^ ¬
- 过度装饰性符号：~ # < > * (仅特定上下文允许)
- 表情符号或表情图标

## 二、结构化属性质量 (30%)

### 2.1 RAI 前 6 项核心属性

| # | 属性 | 检查项 |
|---|------|--------|
| 1 | Product Title | 品牌+核心词+功能/属性/材质+尺寸/规格 |
| 2 | Brand Name | 与备案品牌完全一致（大小写敏感） |
| 3 | Category | 最相关的叶子节点 (Leaf Node)，Node ID 正确 |
| 4 | Bullet Points | 前两点涵盖核心卖点和场景 |
| 5 | Product Type | 精确 PT 值（如 HUMIDIFIER 而非 HOME_DEVICE） |
| 6 | Main Image | 纯白底，像素占比 ≥85%，无水印无附件 |

### 2.2 Band A 属性（转化动力）
- [ ] Size/Color/Material 通过变体关系正确表达
- [ ] Item Weight/Dimensions 与物流实测一致
- [ ] Model Number/Style 准确填写

### 2.3 Band B 属性（体验增强）
- [ ] Special Features 已填写
- [ ] Target Audience 已填写
- [ ] Usage Scenarios 已填写

### 2.4 属性填写完整性
- [ ] 所有与类目相关的下拉属性已填写（不相关的填空格）
- [ ] 可自编字符的属性已填相关关键词
- [ ] Search Terms 已填充
- [ ] 无空值被系统跳过（不确定的填空格而非留空）

## 三、标题质量 (25%) — TQS 逻辑

### 3.1 A 级标题要求（全部满足）
- [ ] 标题完整性评分 ≥ 0.6（标注 ≥3 个属性）
- [ ] 冗余度评分 = 1（无冗余，同词不超过 2 次）
- [ ] 长度 ≤ 200 字符
- [ ] 管道符/分隔符 ≥ 1 个
- [ ] 无特殊字符（! $ ? _ { } ^ ¬）
- [ ] 无限制性短语
- [ ] 长度建议 80-200 字符
- [ ] 以品牌名开头
- [ ] 每个单词首字母大写（介词/连词/冠词除外）
- [ ] 使用阿拉伯数字（2 而非 two）
- [ ] 计量单位缩写（cm, oz, in, kg）

### 3.2 C 级标题（任一触发即为 U 级）
- [ ] 完整性评分 < 0.4（属性 < 2 个）
- [ ] 冗余度 ≤ 0.6（重复词 > 2 次）
- [ ] 长度 > 200 字符
- [ ] 存在受限短语

## 四、变体质量 (20%)

### 4.1 严重缺陷（立即修复）
- [ ] 属性值错误
- [ ] 单位制混用（英制/公制）
- [ ] 变体信息冗余
- [ ] 本地化错误
- [ ] 子项重复

### 4.2 非严重缺陷（逐步改善）
- [ ] 未标准化值
- [ ] 格式不匹配
- [ ] 变体主题错误

### 4.3 基本要求
- [ ] 变体数量 ≥ 2
- [ ] 变体主题属性全部正确且一致
- [ ] 主题属性无重复

### 4.4 变体合理性分析（v1.1.0 新增）

#### 结构概览
- [ ] 识别变体主题（如 COLOR / STYLE / SIZE）
- [ ] 统计子 ASIN 数量
- [ ] 确认 Parent ASIN 存在

#### 维度合理性评估
- [ ] 列出该品类所有可用变体维度
- [ ] 评估当前使用的维度是否为最合理选择
- [ ] 检查是否有遗漏的合理维度（如不同容量、不同功率版本）
- [ ] 判断是否存在应拆分为独立 Listing 而非变体的情况

#### 属性值规范性
- [ ] 各子变体的属性值是否标准化（如颜色值不含技术参数前缀）
- [ ] 属性值是否符合类目枚举
- [ ] 各子变体间的属性值命名是否一致
- [ ] 是否存在冗余或不相关信息混入属性值

#### 家族覆盖度
- [ ] 变体数量是否覆盖产品线主要差异
- [ ] 是否有常见变体缺失（如某颜色/尺寸未上架）
- [ ] A+ 或描述中是否提到了未在变体家族中体现的版本

#### 待确认问题
- [ ] 列出需要用户提供更多信息才能判断的问题
- [ ] 标注数据不足导致的无法判断项

## 五、图片质量 (15%)

### 5.1 基本要求
- [ ] 网站图片数量 ≥ 4 张
- [ ] 无图片质量缺陷
- [ ] 主图有缩放功能（有效像素占比 ≥85%）
- [ ] 主图纯白底 (255,255,255)
- [ ] 无水印、无附件、无文字叠加

### 5.2 加分项
- [ ] IAI (Image Attribute Index) ≥ 0.4
- [ ] 图片 alt 属性完整
- [ ] 图片文件名包含关键词
- [ ] 产品在图片中占比合理

## 六、五点描述 (5%)

### 6.0 BP 前台验证（⚠️ 强制步骤，v1.5.0 强化）
- [ ] **【强制】** 已通过前台 web_reader 抓取确认 BP 数量（严禁仅依赖后台 API）
- [ ] **【强制】** 已提取前台实际展示的 BP 内容
- [ ] 已与后台数据对比，以前台数据为准
- [ ] 若前台抓取失败，已在报告中注明"无法确认前台 BP 数量"

### 6.1 数量要求
- [ ] ≥ 3 条 → 100%
- [ ] ≥ 1 条 → 50%
- [ ] 0 条 → 0%

### 6.1.1 数据来源优先级（v1.2.0 新增）
- [ ] 当前台展示 BP 数量与后台 API 返回不一致时，**以前台抓取数据为准**
- [ ] 后台 API 数据返回不全视为已知 bug，不作为 Listing 缺陷扣分
- [ ] 在分析中注明数据来源差异

### 6.2 质量要求
- [ ] 无违禁关键词
- [ ] 无敏感词
- [ ] 遵循 0-1-3 原则（基础功能 0 遗漏，1 个核心卖点，3+ 魅力卖点）
- [ ] 前两点涵盖核心卖点和使用场景
- [ ] 包含关键属性词

## 七、A+ 页面 (5%)

### 7.0 A+ 前台检测（v1.4.0 新增）
- [ ] **【强制】** 已通过前台 web_reader 检查 "Product description" 模块（结构化内容含子标题）
- [ ] **【强制】** 已通过前台 web_reader 检查 "From the brand" 模块（含图片/视频）
- [ ] 以上任一模块存在即判定 A+ 内容存在，评 100%
- [ ] 若前台抓取失败且后台无标记，标注"数据不足，无法确认"

### 7.1 评分
- [ ] 有 A+ 详情页内容 → 100%
- [ ] 有高级 A+ → 额外加分
- [ ] A+ 页面中的图片文件名包含关键词
- [ ] A+ 内容覆盖详情页中客户反馈的关键信息

- [ ] 核心关键词在 QA 中出现
- [ ] 核心关键词在图片标签中出现
- [ ] 核心关键词在 A+ 页面图片文件名中出现

## 十、整体合规性

- [ ] 内容不存在冲突或误导
- [ ] 无 AI 自动修正风险（不当标点、字符、违规词）
- [ ] 无类目特殊限制违规（医疗/儿童/食品额外要求）
- [ ] 商品信息不是重复页面
- [ ] 变体关系有效
- [ ] 商标信息未被错误修改

> **注意（v1.1.0）：** GPSR (General Product Safety Regulation) 合规评估已排除。安全信息、制造商信息、EU 负责人信息仅作为数据完整性问题评估，不作为合规风险评分。

## 十一、数据采集工具检查（v1.3.0 新增）

### MCP 工具可用性
- [ ] `web_reader` MCP 工具是否可用（用于前台文本数据抓取）
- [ ] `analyze_image` MCP 工具是否可用（用于 A+ 图片模块分析）

### 数据完整性验证
- [ ] A+ 内容是否同时通过文本抓取和图片分析获取（避免遗漏图片类模块）
- [ ] 若仅有文本抓取，报告中是否标注"A+ 内容可能不完整"
- [ ] 前台 BP 数据是否与后台 API 数据对比验证
- [ ] 若工具不可用，是否已要求用户手动补充数据

---

## 十二、HTML 报告输出检查（v1.5.0 新增）

### 模板使用
- [ ] **【强制】** HTML 报告是否基于 `reference/report-template.html` 模板生成
- [ ] CSS 样式是否与模板一致（未自行修改）
- [ ] 17 个 section 是否按模板顺序完整输出

### JSON→Excel 字段解析（用户提供 JSON 时）
- [ ] 是否先输出了字段解析 Excel 表（原文+中文翻译）
- [ ] 字段名格式是否为 `field_name（中文名）`
- [ ] 枚举值是否翻译为中文，自由文本是否保留原文
- [ ] 嵌套字段是否展平为 `parent.child` 格式
- [ ] 输出文件是否保存到用户桌面
