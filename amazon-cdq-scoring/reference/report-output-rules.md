# CDQ v3 报告输出规范

> 本文件定义 CDQ v3 诊断报告的输出格式规则，所有报告必须遵守。

---

## 一、报告格式

### 1.1 输出文件
- 每次诊断同时输出 **MD + HTML** 两种格式
- 文件命名：`CDQ_v3_Report_{ASIN}.md` 和 `CDQ_v3_Report_{ASIN}.html`
- 保存路径：用户桌面（`C:\Users\{username}\Desktop\`）

### 1.2 版本号
- **报告中不出现任何版本号**（如 v1.0、v1.3.0 等）
- 版本信息仅记录在 SKILL.md 的变更日志中

### 1.3 报告结构（按顺序）
1. 基本信息
2. 总分 + 等级
3. 各维度得分汇总表
4. 政策违规检测（U 级风险扫描）
5. 维度 1：结构化属性质量
6. 维度 2：标题质量
7. 维度 3：变体质量
8. 维度 4：产品类型分类
9. 维度 5：图片质量
10. 维度 6：五点描述
11. 维度 7：A+ 页面
12. 高频词检查
13. 整体合规性
14. 缺陷清单
15. 优化优先级建议
16. 预期优化效果（分阶段）
17. 附录：商业表现参考（不直接影响评分）

## 二、数据来源规则

### 2.1 前台优先（⚠️ 强制规则）
- **五点描述 (BP) 数量必须通过前台 web_reader 抓取确认**，严禁仅凭后台 API 数据评分
- 后台 API 仅返回部分 BP 是已知 bug（可能仅返回 1 条而实际有 5 条）
- **评分前必须先执行前台抓取**，若前台抓取失败，报告中必须注明"无法确认前台 BP 数量，评分基于后台数据（可能不完整）"
- 前台已验证存在的内容，状态标记为 ✅，不因后台数据不全而标 ⚠️
- 后台 API 数据不全不作为 Listing 缺陷扣分

### 2.2 数据差异标注
- 前后台数据不一致时，在报告中注明差异（如评论数、标题内容）
- 不重复标注前台已验证通过的问题

### 2.3 Search Terms 字段
- Catalog API 不返回 Search Terms 字段，工具不支持查询
- 报告中标记为 ⚠️（非 ❌），说明"工具不支持查询，请至卖家后台确认是否已填写完整"
- 缺陷清单中优先级为 🟡 中等（非 🔴 严重），因为无法确认实际状态

---

## 三、状态标记规则

### 3.1 ✅ / ⚠️ / ❌ 使用标准
| 标记 | 含义 | 示例 |
|------|------|------|
| ✅ | 合格/已填写/已验证 | BP 前台验证 5 条 ✅ |
| ⚠️ | 需关注/不完整/有风险 | Material 仅填写部分 ⚠️ |
| ❌ | 缺失/错误/未填写 | Search Terms 字段不存在 ❌ |

### 3.2 优先级标记
- 🔴 严重：影响评分 ≥2 分或存在政策风险
- 🟡 中等：影响评分 1~2 分
- 🟢 低：加分项缺失或轻微问题

---

## 四、不输出内容

以下内容 **不在报告中呈现**：

- ~~数据采集工具状态~~（MCP 工具是否可用、web_reader / analyze_image 状态）
- ~~版本号~~（v1.x.0 等）
- ~~GPSR 合规评估~~（仅作为数据完整性问题评估，不作为合规风险评分，footer 中注明"GPSR 合规评估已排除"即可）

## 五、HTML 报告样式

### 5.1 固定模板（v1.5.0 新增）
- **必须使用** `reference/report-template.html` 作为 HTML 报告的固定模板
- 模板包含完整的 CSS 样式和 HTML 骨架（17 个 section 容器 + footer）
- **不允许修改模板中的 CSS 样式和布局结构**
- 只需将 `{{PLACEHOLDER}}` 占位符替换为实际数据内容

### 5.2 变量片段格式规范
以下内容类型动态生成，但必须遵循固定 HTML 格式：

**基本信息行：**
```html
<tr><th style="width:140px;">字段名</th><td>值</td></tr>
```

**状态标记：**
```html
<span class="pass">✅</span>  <!-- 通过 -->
<span class="warn">⚠️</span>  <!-- 警告 -->
<span class="fail">❌</span>  <!-- 失败 -->
```

**优先级标签：**
```html
<span class="tag-red">🔴 严重</span>
<span class="tag-orange">🟡 中等</span>
<span class="tag-green">🟢 低</span>
```

**提示框：**
```html
<div class="alert-red"><h4>标题</h4><p>内容</p></div>
<div class="alert-orange"><h4>标题</h4><p>内容</p></div>
<div class="alert-green"><h4>标题</h4><p>内容</p></div>
```

**引用块：**
```html
<div class="quote"><span class="quote-title">标签</span><br>内容</div>
```

**缺陷行：**
```html
<tr><td><span class="tag-red">🔴 严重</span></td><td>维度</td><td>缺陷类型</td><td>当前状态</td><td>修复建议</td><td>加分</td></tr>
```

**优化阶段列表项：**
```html
<li><strong>关键项</strong> — 说明</li>
```

### 5.3 整体风格
- 单页 HTML，内联 CSS，不依赖外部资源
- 响应式布局，移动端适配
- 配色：蓝白为主，绿/橙/红用于状态标记

---

## 六、报告尾部固定内容

每份报告 Footer 统一为：
```
本报告由 CDQ v3 诊断 Skill 生成
数据来源：后台 Catalog API + 前台 web_reader 抓取 + 图片 analyze_image 分析 | GPSR 合规评估已排除
```
